# 転記処理 完全フロー（実績1入力まで）

> 本ドキュメントは参考2.txt（萩山氏の操作説明録音）および実機HTML検証に基づく。
> 参考文献と実装に乖離がある場合、本ドキュメント（=参考文献）を正とする。

---

## 前提条件

- 転記前にスタッフがカナミックに登録済みであること（参考1.txt参照）
- SmartHR から従業員の資格情報を取得済みであること
- HAM にログイン済みであること

---

## カナミックへ転記される条件（月次シート参照カラム）

| カラム | 名前 | 条件 | 結果 |
|--------|------|------|------|
| S列 | 転記フラグ | 空 | 新規転記対象 |
| S列 | 転記フラグ | `エラー：システム` | リトライ対象 |
| S列 | 転記フラグ | `修正あり` | 再転記対象（先に既存削除） |
| S列+T列 | 転記フラグ+マスタ修正フラグ | `エラー：マスタ不備` + T列=TRUE | マスタ修正後の再転記対象 |
| S列 | 転記フラグ | `転記済み` | 対象外（スキップ） |
| Y列 | 実績ロック | TRUE | 対象外（スキップ） |

## 変更管理シートに入る条件

修正検出は `CorrectionDetector.detectCorrections` で行う。

| カラム | 名前 | 条件 |
|--------|------|------|
| S列 | 転記フラグ | `転記済み` であること |
| C列 | 更新日時 (updatedAt) | V列より後であること |
| V列 | データ取得日時 (dataFetchedAt) | 転記実行時に記録される |

検出条件: **S列 = `転記済み` AND C列(updatedAt) > V列(dataFetchedAt)**

検出後: 看護記録修正管理シートにレコード追加 + 月次シートのS列を `修正あり` に更新

## U列エラー詳細の分類一览

転記エラー時に U列（エラー詳細）に書き込まれるメッセージは、`TranscriptionWorkflow.classifyError` で分類される。  
S列のステータス（`エラー：マスタ不備` / `エラー：システム`）と対応する。

| 分類 | S列ステータス | U列に表示される文言（例） |
|------|----------------|---------------------------|
| 利用者未登録 | エラー：マスタ不備 | 利用者がHAMに登録されていません |
| スタッフ未登録 | エラー：マスタ不備 | スタッフがHAMに登録されていません |
| 医療リハビリ資格制限 | エラー：マスタ不備 | 医療リハビリ：看護師/准看護師は対応不可（理学療法士等のみ） |
| サービスコード未検出 | エラー：システム | サービスコードが見つかりません。HAM設定を確認してください |
| HAMシステム一時障害 | エラー：システム | HAMシステムが一時的に利用できません。時間をおいて再実行してください |
| 画面読み込みタイムアウト | エラー：システム | HAM画面の読み込みタイムアウト。再実行してください |
| 画面遷移エラー | エラー：システム | HAM画面遷移エラー。再実行してください |
| ネットワークタイムアウト | エラー：システム | ネットワークタイムアウト。接続を確認して再実行してください |
| セッション切れ | エラー：システム | セッション切れ。再ログインして再実行してください |
| その他（未分類） | エラー：システム | システムエラー: （元メッセージ先頭80文字） |

**判定の優先順位:** 上から順に元エラーメッセージにキーワードが含まれるかで判定。該当しなければ最後の「その他」として先頭80文字を表示する。

## A. 通常フロー（医療/精神医療/介護通常）

### Step 1: 利用者スケジュール画面へ遷移

```
HAM メインメニュー (t1-2)
  → 訪問看護業務ガイド (k1_1) [act_k1_1]
    → 4-8 利用者スケジュール (k2_1) [act_k2_1]
```

参考2: 「入力する場所はここ4の8利用者スケジュールです」

### Step 2: 年月設定 & 利用者検索 (k2_1)

1. `select[name="searchdate"]` で対象年月を設定（例: `20260201`）
2. 50音ボタン（タ行など）を押すか、検索ボタンを押して利用者を検索
3. 利用者一覧が表示される

参考2: 「年月なのでまずは2月を入れる」「ここで（タ）って押すと他業の登録のあるご利用者様が出てくる」

**セレクタ:**
- 年月: `select[name="searchdate"]`
- 検索: `act_search`
- カナ検索: `xinwork_searchKeyword(form, 'タ', 'act_keyword')`

### Step 3: 利用者を選択 → 月間スケジュール (k2_2)

1. 利用者の行にある「決定」ボタンをクリック
2. → k2_2 月間スケジュールに遷移

参考2: 「ここの月管スケジュールっていうところの決定をします」

**セレクタ:**
- 決定ボタン: `input[name="act_result"][value="決定"]`
- onclick: `submitTargetFormEx(this.form, 'k2_2', careuserid, '{patientId}')`
- 患者IDの抽出: `careuserid, '{ID}'` パターン

### Step 4: 追加ボタン → スケジュール追加 (k2_3)

1. k2_2 で「追加」ボタンをクリック

参考2: 「追加ボタン押します。追加ボタン押すと時間が入れれるようになります」

**セレクタ:**
- 追加: `act_addnew` (setLockCheck=true)
- hiddenFields: `editdate = '{YYYYMMDD}'`（日付指定）

### Step 5: 時間設定 (k2_3)

1. **日付**: 上部の日付選択（月のプルダウンで01日～）
2. **開始時間帯**: `select[name="starttype"]`
   - `1` = 昼間（6:00-18:00）
   - `2` = 夜間・早朝
   - `3` = 深夜
3. **開始時刻**: `select[name="starttime0"]`(時) + `select[name="starttime1"]`(分)
4. **時間区分**: `select[name="timetype"]`（専務確認済み統一ルール 2026-02-26）
   - `20` = 20分未満（差分 0~20分）
   - `30` = 30分未満（差分 21~30分）
   - `60` = 1時間未満（差分 31~60分）
   - `90` = 1時間30分未満（差分 61~90分）
   - `91` = 1時間30分以上（差分 91分~）
5. **終了時刻**: timetype選択で自動入力される → **手動修正しない**（修正するとエラーになる）

参考2:
- 「1日にする → 昼間選ぶ → 10時からにして → 30分未満ってすると → ここが勝手に入る（10時29分）」
- ~~「でも29分じゃなくて30分なのでこう書き換えます」~~ → **専務確認済み: 手動修正不要（2026-02-26）**
- 「30分ジャストだったら30分未満でつけて、60分だったら1時間未満でつけてください。医療も共に一緒で大丈夫です」

7. **次へ** → k2_3a

**セレクタ:**
- 次へ: `act_next`

### Step 6: サービスコード選択 (k2_3a)

#### 6a. 保険種別の確認/切替

k2_3a 画面上部に3つのタブボタン:
- `訪問看護` (showflag=1, 介護保険)
- `予防訪問看護` (showflag=2, 予防)
- `訪問看護療養費` (showflag=3, 医療保険)

**分岐ルール（参考2より）:**
- 利用者が**医療保険のみ**登録 → 医療タブが最初から選択済み、切替不要
- 利用者が**介護保険のみ**登録 → 介護タブが最初から選択済み、切替不要
- 利用者が**両方登録** → 支援区分に応じてタブを切り替える必要あり
  - 支援区分1=「医療」or「精神医療」→ `訪問看護療養費`タブ (showflag=3)
  - 支援区分1=「介護」→ `訪問看護`タブ (showflag=1) or `予防訪問看護`タブ (showflag=2)

参考2: 「介護保険も医療保険も登録されてるご利用者様...ここのボタンで支援区分が精神医療なのでこのボタンを押す」「医療保険だけの人だったらもう最初っから表示されてるから押さなくていい」

**セレクタ:**
- 切替: `form.showflag.value = '{flag}'; form.doAction.value = 'act_change'; form.submit()`

#### 6b. 資格選択（医療保険 showflag=3 の場合のみ）

サービスコード検索エリアにラジオボタン:
- `● 看護師等`
- `○ 准看護師`
- `○ 理学療法士等`
- `○ 慢性疾患...`
- その他チェックボックス: 緊急、長時間、乳幼児、複数名訪問、複数名訪問(二)

**従業員の資格を確認し、該当するラジオを選択してから「検索」ボタンを押す。**

参考2:
- 「支援に入った人の資格を見て...看護師だったら看護師等、准看護師だったら准看護師等、理学療法士等にチェックを入れなきゃいけない」
- 「検索って押すんですよ。それ押すとここさっきと変わった」
- 看護師で検索 → 「訪問看護基本療養費（Ⅰ・Ⅱ）」
- 准看護師で検索 → 「訪問看護基本療養費（Ⅰ・Ⅱ）の准」に変わる
- 理学療法士で検索 → 「理学療法士等」に変わる

#### 6c. サービスコード選択

検索結果のラジオボタンから適切なものを選択:

**医療+通常** (参考2 実機確認):
| 資格 | 選択するコード | HAM radio value |
|------|--------------|----------------|
| 看護師 | 訪問看護基本療養費（Ⅰ・Ⅱ） | `93#1001` |
| 准看護師 | 訪問看護基本療養費（Ⅰ・Ⅱ）准 | 要検証 |
| 理学療法士等 | 訪問看護基本療養費 理学療法 | 要検証 |

**精神医療+通常** (参考2):
| 資格 | 選択するコード | HAM radio value |
|------|--------------|----------------|
| 看護師 | 精神科訪問看護基本療養費（Ⅰ・Ⅲ） | `93#1225` |
| 准看護師 | 精神科訪問看護基本療養費（Ⅰ・Ⅲ）准 | 要検証 |

参考2: 「精神医療でしたよね。なのでこっちです。精神科訪問看護基本療養費」

**医療+リハビリ** (参考2):
- **看護師/准看護師が支援に入っていた場合 → エラー！転記しない**
- 理学療法士/作業療法士/言語聴覚士のみ可

参考2: 「医療リハビリのやつで看護師と准看護師どっちかが支援に入ってたら転記はさせずにエラー」「理学療法士/作業療法士/言語聴覚士の時のみ」

**介護+通常** (参考2):
- 要介護1-5 → `訪問看護`タブ (showflag=1)
- 要支援1-2 → `予防訪問看護`タブ (showflag=2)
- 看護師/准看護師 → 通常フロー（追加で入力）
- 理学療法士等 → **I5フロー（訪看I5入力）に切替が必要**

参考2: 「介護の通常で...看護師か准看護師が入ってたら1番で記録。ただ理学療法士等が入っていたらI5でつけなきゃいけないので戻らなきゃいけない」

**セレクタ:**
- サービスコード radio: `input[name="radio"]`
- 値: `{servicetype}#{serviceitem}`（例: `93#1001`）
- 選択後 hidden 設定: `form.servicetype.value`, `form.serviceitem.value`
- 検索ボタン: (ラジオ選択 + テキストマッチで対応)
- 次へ: `act_next`

### Step 7: 確認 → 決定 (k2_3b)

1. 内容を確認
2. 「決定」ボタンをクリック → k2_2 に戻る
3. **予定のみ**が作成された状態（実績はまだ入力不可）

参考2: 「決定で記録が入ります。予定、予定が経っただけ。まだ実績は作れてない」

**セレクタ:**
- 決定: `act_do`

### Step 8: スタッフ配置 (k2_2 → k2_2f)

1. k2_2 に戻ると、新しく追加された行に「配置」ボタンがある
2. 「配置」ボタンをクリック → k2_2f スタッフ配置画面

参考2: 「この予定を作ったら次ここのスタッフ編集で配置ってありますよね。配置を押します」

**セレクタ (k2_2):**
- 配置ボタン: `input[name="act_modify"][value="配置"]`
- onclick: `submitTargetFormEx(this.form, 'act_modify', assignid, '{assignId}')`
- 新規行の特定: 日付一致(`X日`形式) + スタッフ未配置の行

### Step 9: スタッフ選択 (k2_2f) — 2段階操作

#### Stage 1: 配置ボタンクリック

k2_2f 初期画面: 勤怠時間設定 + 配置ボタン（従業員リストはまだ表示されない）

- `select[name="newstarthour/starthour"]` / `select[name="newstartminute/startminute"]` — 自動入力済
- `select[name="newendhour/endhour"]` / `select[name="newendminute/endminute"]` — 自動入力済
- スタッフ欄: 空 or デフォルト（例: システム カンリシャ）
- **配置ボタン**: `submitTargetForm(document.forms[0], 'act_select')` ← **これをクリック**

#### Stage 2: 従業員選択リスト

配置ボタンクリック後、**従業員選択リストが表示される**:

**スタッフ一覧テーブル**
| 列 | 内容 |
|----|------|
| 従業員番号 | 例: `1793` |
| スタッフ氏名 | 例: `荒垣 久美子`（全角スペース区切り） |
| 性別 | 男/女 |
| 資格 | 看護師 / 准看護師 / 理学療法士 / 作業療法士 / 言語聴覚士 |
| 訪問する職種 | hidden `name="jobtype"` (03=看護師, 04=理学療法士, 05=作業療法士, 06=言語聴覚士, 07=准看護師) |
| 確認ボタン | `onclick="return haiji(this, '{helperId}', '{name}', 1)"` |
| 選択ボタン | `onclick="return choice(this, '{helperId}', '{name}', 1)"` |

**操作手順:**
1. スタッフ一覧から記録者の名前を探す
2. 「選択」ボタンをクリック → choice() が helperid を設定しフォーム送信
3. → k2_2 に戻る

参考2: 「これ入れて選択を押します。で、決定で上書き保存」

**注意:**
- 複数資格を持つスタッフ（例: 萩山氏 = 看護師+理学療法士+作業療法士+言語聴覚士）は職種が `select` ドロップダウンになる
- 既に別の予定で使用中のスタッフは `class="busy"` で `disabled` になる

### Step 10: 上書き保存（1回目）

1. k2_2 に戻ると、スタッフが配置された状態
2. **上書き保存**をクリック

参考2: 「上書き保存すると戻ると配置されましたよね、従業員が」

**セレクタ:**
- 上書き保存: `act_update` (setLockCheck=true)

### Step 11: 実績入力

1. スタッフ配置後、**実績欄に入力可能になる**
2. 実績に `1` を入力（手動 or 全1ボタン）

参考2:
- 「さっきまでここ選べなかったのに選べるようになってるの分かります？ここに1が立てれるようになるんですよ」
- 「ここの全っていうボタンがあるじゃないですか。これ押すとここにある配置がされた予定に全部1を立てることができる」

**セレクタ:**
- 実績 input: `input[name="results"]` (type="text", size=2, maxlength=1)
- 全1ボタン: `input[name="act_chooseall"]` onclick `checkAllAndSet1('results')`
- 各 results の隣: `input[name="assignids"]` (hidden, value=assignid)

### Step 12: 上書き保存（2回目・最終）

1. 全1 or 手動1入力後、再度「上書き保存」
2. これで**予定と実績**が確定

参考2: 「全（1）立てたらこれまた上書き保存しなきゃいけない。するとこれで予定と実績が入ります」

**セレクタ:**
- 上書き保存: `act_update` (setLockCheck=true)

## B. I5フロー（介護+リハビリ専用）

参考2: 「支援区分が介護になってリハビリっていう風に支援区分2がなってるやつはこの訪看I5ってやつで入力が必要」「I5っていうボタンを押すのは介護リハビリの人だけ」

### 違い

| 項目 | 通常フロー | I5フロー |
|------|----------|---------|
| 入力ページ | k2_3 → k2_3a → k2_3b | k2_7_1 |
| 開始ボタン | 追加 | 訪看I5入力 |
| 時間単位 | 自由 (timetype) | 20分単位 (1回/2回/3回) |
| 保険選択 | showflag切替 | 介護/予防 選択ボタン |
| 介護度判定 | 不要(タブで判定) | 必須(要介護1-5→介護, 要支援1-2→予防) |

### I5 時間ルール (参考2)

- 20分 → 1回
- 40分 → 2回
- 60分 → 3回

参考2: 「20分が1回って思って。40分だったら2回、60分だったら3回」「開始時刻を入れたらここ2回っていうボタンを押す。するとここが入るけど（終了時刻が）」

## D. 修正レコードの再転記フロー

既に転記済みのレコードが修正された場合（転記フラグ=「修正あり」）、
**Kanamic 上の既存レコードを先に削除してから、修正後のデータで再転記する**必要がある。

### 削除フロー

1. k2_1 で利用者検索 → k2_2 月間スケジュールを開く
2. 対象の日付・時間のスケジュール行を特定する
   - 日付（`X日`）+ 時間（`HH:MM ～ HH:MM`）+ サービス種類でマッチ
3. その行の「削除」ボタンをクリック
   ```html
   <input name="act_delete" type="button" value="削除" 
          onclick="confirmDelete('{assignid}', '{record2flag}');">
   ```
4. 確認ダイアログが表示される → 承認
5. 上書き保存（削除反映にも上書き保存が必要）
6. 削除完了後、通常の転記フロー（Step 4 の追加から）で再転記

### 特定方法

既存レコードの特定は以下の情報で行う:
- **利用者名**: 同一患者
- **日付**: 同一日（`X日` 表示）
- **時間**: 同一時間帯（`HH:MM ～ HH:MM`）
- **サービス種類**: テキスト一致
- **assignid**: 行のボタン onclick から抽出

### 注意事項
- 削除時も**上書き保存が必要**（参考2: 「削除の時もちなみに上書き保存が必要」）
- record2flag が `1` の場合は記録書Ⅱが存在するため、先に記録書Ⅱを削除する必要がある

## E. 重要な注意事項（参考2より）

1. **お知らせポップアップ**: HAM 画面に注意やお知らせが先に表示されることがある → 閉じてから操作
2. **終了時刻の自動入力**: timetype 選択時に自動で入る → **手動修正しない**（専務確認済み 2026-02-26）
3. **上書き保存は2回必要**: 配置後に1回、全1入力後にもう1回
4. **同名同姓**: 被保険者番号で突合。なければ名前検索のみ
5. **削除時も上書き保存が必要**
6. **介護+通常で理学療法士等**: I5フローに切り替える必要あり（通常の追加ではない）
7. **看護師と准看護師の両方を持つ場合**: 看護師を優先（HAMでは片方しかチェックできない）
8. **修正レコード再転記**: 既存レコードを先に削除（confirmDelete）→ 上書き保存 → 再転記
