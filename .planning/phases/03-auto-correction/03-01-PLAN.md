**Method 1: `retranscribeRecords()`** -- D-06 dedicated re-transcription method. This is a SEPARATE path from processLocation(). It directly calls processRecord() for each target record without re-fetching records, re-logging in, re-checking staff, etc.

```typescript
/**
 * 対象レコードのみを再転記する専用メソッド (D-06)
 * processLocation() の転記ループとは別のパスで、指定レコードのみを処理する。
 * processLocation() の re-login、CSV再読み込み、スタッフ事前チェック等をスキップし、
 * 既存の processRecord() を直接呼び出す軽量な再転記パス。
 */
private async retranscribeRecords(
  records: TranscriptionRecord[],
  nav: HamNavigator,
  sheetId: string,
  tab?: string,
): Promise<{ succeeded: string[]; failed: string[]; nav: HamNavigator }>
```

Implementation:
1. Track arrays: `succeeded: string[]`, `failed: string[]`
2. For each record in records:
   - Wrap in try-catch (D-09):
     - Call `await this.processRecord(record, nav, sheetId, tab)` with the same withRetry + 5-minute timeout pattern used in processLocation's transcription loop (lines 302-314)
     - On success: add recordId to `succeeded`
     - Log: `logger.info(\`[自動修正] 再転記完了: ${record.patientName} ${record.visitDate}\`)`
   - Catch: log error, add recordId to `failed`, recover via `nav = await this.tryRecoverToMainMenu(nav)`
3. Return `{ succeeded, failed, nav }`

Key difference from processLocation: No record fetching, no staff pre-check, no CSV download, no lock handling, no correction sheet sync. Just processRecord() calls for the given records.

**Method 2: `runAutoCorrection()`** -- Orchestrates the full correction cycle (D-01). Self-contained: handles deletion, reset, re-transcription, re-verification, AND result recording (FIX-03).

```typescript
/**
 * 自動修正: 不一致レコードの削除 -> リセット -> 再転記 -> 再検証 -> 結果記録 (FIX-01, FIX-02, FIX-03)
 * 1サイクルのみ実行 (D-08)。例外発生時はそのレコードをスキップして続行 (D-09)。
 * FIX-03: 再検証結果のAB/AC列書き込みもこのメソッド内で完結する。
 */
private async runAutoCorrection(
  location: SheetLocation,
  mismatches: VerificationMismatch[],
  records: TranscriptionRecord[],
  nav: HamNavigator,
  dryRun: boolean,
  tab?: string,
): Promise<{ corrected: string[]; failed: string[]; nav: HamNavigator }>
```

Implementation:
1. Build a recordId -> TranscriptionRecord map from `records` for quick lookup.
2. Separate mismatches into two groups per D-03/D-04:
   - `toDelete`: mismatches where `!mm.missingFromHam` (time/service/staff -- need HAM deletion)
   - `toResetOnly`: mismatches where `mm.missingFromHam === true` (just reset T-column)
3. Track arrays: `corrected: string[]`, `failed: string[]`, `allResetRecords: TranscriptionRecord[]`
4. **Phase A - Delete (D-03):** For each `toDelete` mismatch:
   - Find the TranscriptionRecord by recordId from the map
   - If record has no `hamAssignId`, log warning "assignId不明 -- 自動修正スキップ" and add to `failed`, continue
   - Wrap in try-catch (D-09):
     - Call `this.deleteHamRecord(nav, record)` -- if returns false, add to failed, continue
     - Call `this.sheets.resetForRetranscription(location.sheetId, record.rowIndex, tab)`
     - Add record to `allResetRecords`
     - Log: `logger.info(\`[自動修正] 削除完了: ${record.patientName} ${record.visitDate}\`)`
   - Catch: log error, add to `failed`, recover via `nav = await this.tryRecoverToMainMenu(nav)`
5. **Phase B - Reset only (D-04):** For each `toResetOnly` mismatch:
   - Find record, wrap in try-catch
   - Call `this.sheets.resetForRetranscription(location.sheetId, record.rowIndex, tab)`
   - Add record to `allResetRecords`
   - Log: `logger.info(\`[自動修正] ステータスリセット(missingInHam): ${record.patientName} ${record.visitDate}\`)`
   - Catch: add to failed
6. **Phase C - Re-transcribe (D-06):** If `allResetRecords.length > 0` and not dryRun:
   - Call `this.retranscribeRecords(allResetRecords, nav, location.sheetId, tab)` (D-06: dedicated method, NOT processLocation recursion)
   - Update nav from result: `nav = retranscribeResult.nav`
   - Add retranscribeResult.succeeded to `corrected`
   - Add retranscribeResult.failed to `failed`
7. **Phase D - Re-verify and record results (FIX-03):** If `corrected.length > 0`:
   - Re-fetch fresh records: `const freshRecords = await this.sheets.getTranscriptionRecords(location.sheetId, tab)`
   - Call `await this.runVerification(location, freshRecords, tab)` -- this writes AB/AC columns for re-verified records automatically (the existing runVerification already writes to Sheets)
   - Log re-verification completion: `logger.info(\`[自動修正] 再検証完了\`)`
8. **Phase E - Record failed corrections (FIX-03):** For each recordId in `failed`:
   - Find the record from the map
   - Write error to AC column:
     ```typescript
     try {
       await this.sheets.writeVerificationStatus(
         location.sheetId,
         record.rowIndex,
         new Date().toISOString().replace(/\.\d{3}Z$/, ''),
         'auto_correction_failed',
         tab,
       );
     } catch (e) {
       logger.warn(\`修正失敗ステータス書き込みエラー: ${(e as Error).message}\`);
     }
     ```
9. Return `{ corrected, failed, nav }`
